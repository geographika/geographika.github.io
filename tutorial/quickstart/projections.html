8. Projections¶

MapServer can output maps in many different projections.

Projections are defined using the PROJECTION block, and also using Map Metadata (in the WEB Object):

Your mapfile should look like the one here - projections.map.

Projection codes can be found at http://epsg.io/

Projections are stored in C:\ms4w\proj\nad

8.1. Example¶

You should be able to use the appropriate HTML and mapfiles to create the example below using your local MapServer.

Clicking on a radio button requests the same data in a different projection.

  • WGS84 (EPSG:4326) - this is the World Geodetic System 1984, used by GPS
  • Web Mercator (EPSG:3857) - this is the projection used by all the popular online mapping sites such as Google Maps, OpenStreetMap and Bing
  • World Mollweide - is an equal-area, pseudocylindrical map projection
Mapserver OL3 viewer

WMS 1.3.0 Only EPSG, AUTO2, CRS currently supported

Robinson projection not supported in proj4js - https://github.com/proj4js/proj4js/issues/161 Can see main ones in https://github.com/proj4js/proj4js/tree/master/lib/projections

This needs to be in the same units as the data or, if specifying a different output projection, in the same units as the output projection.

You can use the utility ogrinfo, which is part of the GDAL/OGR library package, to get the extent of a particular Shapefile dataset (or other supported vector formats). Here is the command I used to get the extent for this example:

ogrinfo -al -so states_ugl.shp

This returned the following output:

Layer name: states_ugl
Geometry: Polygon

You can also use ArcView or an open source GIS packages–QGIS, Thuban, etc.

Feel free to change the values of EXTENT to get a better understanding of how it changes your map.
